CI/CD

CI and CD are two terms used very frequently in devops. CI stands for continuous integration while CD stands for continuous deployment or delivery. It is a method used to deliver apps to customers frequently by introducing automation into the software development cycle.

Continuous Integration

This is a practise where developers merger their code into the main branch as often as possible. The changes go through automated testing to ensure that the new code does not break existing code.

Advantages of Continuous integration
  1. Developers are working with the most recent code
  2. There are fewer code branches
  3. Small changes and fixes do not need to be merged
  4. No consistency problems between branches
  5. More reliability because there are small regular changes rather than infrequent larger changes

Continuous Deployment

Continuous deployment is a strategy in software development where code changes to an application are released automatically into the production environment. This automation is driven by a series of predefined tests. Once new updates pass those tests, the system pushes the updates directly to the software's users.

Advantages of continuous deployment
  1. Increased production speed
  2. Leads to faster iteration and faster learning
  3. Leads to better business outcomes
